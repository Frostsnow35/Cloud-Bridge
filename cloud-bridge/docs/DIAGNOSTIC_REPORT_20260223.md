# 系统诊断报告 (System Diagnostic Report)
**日期**: 2026-02-23  
**范围**: 详情展示、数据一致性、数据持久化  

## 1. 核心问题摘要
系统当前存在**数据源文件缺失**导致的数据加载失败问题，同时**前后端字段定义不一致**，导致详情页部分内容为空或只能显示硬编码的死数据。模拟数据生成脚本未覆盖所有业务字段，导致演示效果打折。

## 2. 数据一致性与完整性问题 (Data Consistency)

### 2.1 数据源缺失 (Critical)
后端启动时尝试加载的 JSON 数据文件在磁盘上**不存在**，导致无法初始化模拟数据。
- **缺失文件**: `backend/data/smart_achievements.json`
- **缺失文件**: `backend/data/smart_demands.json`
- **影响**: 系统重启后，若数据库被重置，将无数据可用；或数据量仅为个位数（手动添加的数据）。

### 2.2 实体字段与生成逻辑不匹配
后端实体 (`Entity`) 定义了丰富字段，但数据生成脚本 (`generate_smart_data.py`) **未生成**这些字段，导致数据库中存在大量 `null` 值。

| 实体 (Entity) | 字段 (Field) | 后端定义 | 生成脚本状态 | 前端展示影响 |
| :--- | :--- | :--- | :--- | :--- |
| **Achievement** | `applicationCases` (应用案例) | ✅ 有 | ❌ **缺失** | 显示 "暂无案例展示" |
| **Achievement** | `patentInfo` (专利信息) | ✅ 有 | ❌ **缺失** | 显示 "暂无知识产权信息" |
| **Achievement** | `image` (封面图) | ❌ **无** | ❌ **无** | 强制使用默认占位图 (Unsplash) |
| **Demand** | `contactName` (联系人) | ✅ 有 | ❌ **缺失** | 无法联系或显示空白 |
| **Demand** | `institution` (机构名称) | ✅ 有 | ❌ **缺失** | 显示 "发布企业" (默认值) |

### 2.3 前端硬编码问题 (Hardcoded UI)
前端为了演示效果，将部分本应动态获取的数据**写死**在代码中，导致所有需求/成果看起来都一样。

- **需求详情页 (`DemandDetail.vue`)**:
  - **预期目标**: 写死为列表 ("满足国家标准", "周期6个月" 等)，忽略了不同需求的差异。
  - **合作方式**: 写死为 "技术转让 / 合作开发"，后端其实有 `Type` 字段但未正确映射。
- **成果详情页 (`AchievementDetail.vue`)**:
  - **应用场景**: 文本部分写死，仅动态插入了领域名称。

## 3. 数据持久化现状
- **数据库**: 使用 H2 文件数据库 (`cloud_bridge.mv.db`)。
- **状态**: 正常。只要不删除该文件，数据是持久的。
- **问题**: 用户感知的"数据丢失"实际上是因为**初始化脚本未能运行**（因缺文件），且旧数据可能被手动清理或覆盖。

## 4. 建议修复方案

1.  **增强数据生成器**:
    - 修改 `generate_smart_data.py`，补充生成 `applicationCases`, `patentInfo`, `institution`, `contactName` 等字段。
    - 增加 `image` 字段的模拟（即使是随机 URL）。
    - 立即运行脚本重新生成 `smart_achievements.json` 和 `smart_demands.json`。

2.  **后端实体微调**:
    - 给 `Achievement` 增加 `image` 字段，或在前端完善图片映射逻辑。

3.  **前端去硬编码**:
    - 将 `DemandDetail.vue` 中的 "预期目标" 改为显示 `description` 的一部分，或在后端增加 `expectedGoals` 字段。
    - 移除硬编码的 "应用场景" 描述，改为直接显示数据。

4.  **重启初始化**:
    - 确保 `DataInitializer` 能正确读取新生成的 JSON 文件。
