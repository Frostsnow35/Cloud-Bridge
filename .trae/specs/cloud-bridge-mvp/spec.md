# 云转桥 (Cloud Bridge) MVP Spec

## Why
当前科技成果转化面临“信息不对称”、“匹配效率低”、“信任缺失”三大痛点。本项目旨在构建一个基于垂域AI和区块链技术的智能匹配与转化平台，以解决上述问题。本MVP版本聚焦于核心价值验证，即“可信供需对接”、“智能匹配引擎”与“关键环节存证”。

## What Changes
本MVP版本将实现以下核心模块：

### 1. 用户与认证中心 (User Center)
- **角色管理**：支持科研人员（供给侧）、企业（需求侧）、技术经理人（中介侧）三类角色注册与登录。
- **实名认证**：基础的企业/个人实名认证流程（MVP阶段可简化为后台审核）。

### 2. 供需发布与管理 (Supply & Demand Management)
- **成果发布**：科研人员可发布科技成果，包括技术参数、应用场景、成熟度等。
- **需求发布**：企业可发布技术需求或“揭榜挂帅”任务。
- **信息审核**：后台管理员对发布信息进行合规性与真实性审核。

### 3. AI智能匹配引擎 (AI Matching Engine)
- **语义理解**：集成ChatGLM-6B模型，对需求描述进行语义分析，提取关键技术实体。
- **知识图谱构建**：基于Neo4j构建基础的产业知识图谱（MVP阶段聚焦于单一垂直领域，如生物医药或新材料）。
- **智能推荐**：基于“语义相似度 + 图谱关联”算法，为需求方推荐匹配的成果/专家。

### 4. 区块链存证服务 (Blockchain Evidence Service)
- **存证上链**：对接FISCO BCOS联盟链，对关键动作（如技术交底、需求确认、合作意向书）生成哈希并上链。
- **存证核验**：提供存证证书生成与链上数据核验功能，确保数据不可篡改。

## Impact
- **核心业务流程**：打通从“需求发布 -> 智能匹配 -> 意向确认 -> 存证确权”的最小闭环。
- **技术架构验证**：验证“Spring Boot + Vue3 + Neo4j + ChatGLM + FISCO BCOS”混合架构的可行性。

## ADDED Requirements

### Requirement: 可信供需市场
- **WHEN** 用户发布需求/成果 **THEN** 系统进行自动/人工审核，审核通过后公开展示。
- **WHEN** 用户浏览详情 **THEN** 系统展示发布者的认证状态与信用等级（基础版）。

### Requirement: AI智能匹配
- **WHEN** 企业发布技术需求 **THEN** 系统自动分析需求文本，从库中匹配最相关的Top 5成果。
- **WHEN** 用户查看匹配结果 **THEN** 系统提供简要的推荐理由（基于关键词或图谱路径）。

### Requirement: 全流程存证
- **WHEN** 双方达成合作意向或传输关键文档 **THEN** 系统自动计算文件哈希，调用区块链合约进行存证。
- **WHEN** 用户需要自证 **THEN** 系统生成包含链上交易哈希的电子存证证书。

## MODIFIED Requirements
无（新项目开发）

## REMOVED Requirements
- **资源智能导航 (Resource Navigation)**：MVP阶段暂不包含复杂的政策/资金/设备导航功能，留待后续迭代。
- **高级数据分析看板**：MVP阶段仅提供基础的业务数据统计，暂不提供深度产业洞察看板。
- **复杂的司法对接**：MVP阶段仅实现链上存证，暂不对接公证处/法院的实时接口。
