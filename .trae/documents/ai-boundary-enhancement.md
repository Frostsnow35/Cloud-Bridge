# AI Assistant Boundary & Intent Enhancement Spec

## Why
The current AI assistant lacks clear boundaries and comprehensive intent recognition. It fails to handle user private data navigation (e.g., "my applications") and does not properly refuse out-of-scope questions (e.g., "compare these two achievements"). The user requires a strict but polite boundary definition, ensuring the AI serves as a navigator/guide rather than a decision-maker or general chatbot.

## What Changes
- **Backend (`AIService.java`)**:
    - **Prompt Engineering**:
        - Define strict role: "Cloud Bridge Tech Transfer Assistant".
        - Add **Negative Constraints**: Refuse to analyze/compare achievement quality; Refuse general topics (weather, politics).
        - Add **Disclaimer**: For terminology explanations, append "Generated by AI, not official interpretation."
        - Add **Polite Refusal**: Redirect out-of-scope queries back to platform business (e.g., "I can't compare, but I can help you find more...").
    - **Intent Expansion**:
        - Add `NAVIGATE` rules for private pages: "My Needs", "My Achievements", "My Applications", "My Collections".
    - **Logic Update**:
        - Detect if the user is asking for "comparison" or "analysis of quality" and force a `REFUSE` or `CHAT` intent with a guided refusal message.

## Impact
- **Backend**: `AIService` prompt logic will be significantly updated.
- **Frontend**: No major code changes expected, as the contract (`intent`, `reply`, `action`) remains consistent. The AI's *replies* will just be higher quality.

## ADDED Requirements
### Requirement: Private Data Navigation
The AI SHALL recognize intents to view user-specific data and return `NAVIGATE` actions to:
- `/profile/my-needs` (My Needs)
- `/profile/my-achievements` (My Achievements)
- `/profile/applications` (My Applications/Approvals)
- `/profile/collections` (My Collections)

### Requirement: Boundary Enforcement
The AI SHALL refuse to answer:
- Comparative analysis of achievements (e.g., "Which one is better?").
- General open-domain questions (e.g., "Weather today").
- **Refusal Style**: Polite and guiding (e.g., "I focus on tech transfer navigation. I can help you search for...").

### Requirement: Terminology Explanation
WHEN the user asks for a definition (e.g., "What is graphene?"),
THEN the AI SHALL provide a brief explanation based on internal knowledge AND append the disclaimer: "（由AI生成，不代表对任何成果的解释）".
