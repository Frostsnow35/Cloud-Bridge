# 本地测试与一键启动配置指引

本文档旨在帮助开发人员或评审专家在没有预装 Java/Node.js/Maven 环境的 Windows 主机上，快速部署并运行“云桥”平台。

## 1. 前置要求

唯一需要安装的软件是 **Docker Desktop**。

*   **下载地址**: [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
*   **安装说明**: 
    1. 下载并运行安装程序。
    2. 安装完成后，启动 Docker Desktop。
    3. 确保 Docker 引擎状态为 "Running"（绿色图标）。

## 2. 快速启动（一键部署）

我们提供了一键启动脚本，自动处理环境构建、依赖下载和服务编排。

### 步骤
1.  进入项目根目录（例如 `E:\数据要素大赛作品\`）。
2.  双击运行 **`一键启动.bat`**。
3.  脚本将执行以下操作：
    *   检查 Docker 环境。
    *   自动构建后端（Java Spring Boot）和前端（Vue3）的 Docker 镜像。
    *   启动 PostgreSQL、Neo4j、Elasticsearch 等依赖服务。
    *   等待服务初始化完成。
    *   **自动打开浏览器**访问 `http://localhost`。

> **注意**：首次运行时，Docker 需要下载基础镜像（JDK, Nginx, Database等）并编译项目代码，视网络情况可能需要 **5-15 分钟**，请耐心等待。后续启动将非常快（<1分钟）。

## 3. 停止服务

测试完成后，为释放系统资源，请双击运行根目录下的 **`停止服务.bat`**。该脚本将安全地停止并移除所有相关容器。

## 4. 常见问题排查

### Q1: 脚本提示“未检测到 Docker 环境”
*   **解决**: 请确认 Docker Desktop 已安装并正在运行。可以在命令行输入 `docker -v` 检查。

### Q2: 启动过程中卡在 `Downloading ...` 或 `Building ...`
*   **解决**: 这是正常现象，首次构建需要下载大量依赖。请确保网络连接正常。

### Q3: 浏览器打开显示 `502 Bad Gateway` 或无法连接
*   **解决**: 后端服务可能仍在初始化中（尤其是数据库和 Elasticsearch）。请等待 1-2 分钟后刷新页面。

### Q4: 如何查看详细日志？
*   **解决**: 
    *   查看后端日志：`docker logs -f cloud-bridge-backend`
    *   查看前端日志：`docker logs -f cloud-bridge-frontend`

## 5. 开发者模式（手动启动）

如果您已安装开发环境（JDK 17+, Maven 3+, Node.js 18+），也可以使用传统方式启动：

1.  **启动中间件**（可选，或使用 Docker 仅启动中间件）：
    ```bash
    docker compose up -d postgres neo4j elasticsearch
    ```
2.  **启动后端**：
    ```bash
    cd cloud-bridge/backend
    mvn spring-boot:run
    ```
3.  **启动前端**：
    ```bash
    cd cloud-bridge/frontend
    npm run dev
    ```
4.  访问 `http://localhost:5173` (前端开发服) 或 `http://localhost:8080` (后端 API)。
